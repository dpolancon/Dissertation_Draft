# Crisis Theory Applied to Chile - Obsidian Asset Pack

This package is a **vault fragment** designed for Obsidian. It is organized to help build a research system rather than a loose folder of notes.

## What is inside

- `00_Hub/` → entry points and maps of content
- `01_Maps/` → thematic navigation notes
- `02_Concepts/` → core theoretical concepts
- `03_Authors_and_Schools/` → author and school summaries
- `04_Chile_Cases/` → country-specific historical and analytical notes
- `05_Periodization_and_Timelines/` → historical sequencing
- `06_Method_and_Questions/` → research design, framing, and analytical strategy
- `07_Ledgers/` → reference ledgers, source functions, and synthesis trackers
- `08_Templates/` → reusable note templates
- `09_Workbench/` → temporary drafting and synthesis zone

## Suggested use in Obsidian

1. Drop this folder into your vault.
2. Open `00_Hub/00_Home.md`.
3. Expand the pack by creating reading notes and linking them into:
   - concept notes,
   - author notes,
   - Chile case notes,
   - and the master ledger.
4. Treat `01_Maps/` as the navigation layer and `07_Ledgers/` as the control layer.

## Design principle

The pack distinguishes four things:

1. **Concepts** — crisis, overaccumulation, regulation, external constraint, profitability, realization, etc.
2. **Authors / schools** — Marxian, regulationist, dependency, Kaleckian, world-systems, Chilean political economy.
3. **Historical cases** — concrete Chilean conjunctures.
4. **Research operations** — questions, sources, note templates, comparison grids.

## Naming conventions

- Prefix folders numerically to preserve order.
- Use short file names with underscores.
- Use wiki links aggressively.
- Put metadata in YAML frontmatter when helpful.
